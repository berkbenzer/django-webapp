    The aim here is to draw attention to two separate scenarios. As a starting scenario, we will deploy our Django web program to a single AWS EC2 instance.

    In our second scenario, we will create a pipeline to Kubernetes Cluster using IaaC tools and test HA.

    We will publish our infrastructure design, Django-Webapp, which runs in Linux-based Kubernetes Cluster on AWS, with Nginx Reverse Proxy. And with Jenkins we will be publishing new Version of our Web App To check Foreman-Katello the patch status of our infrastructure. We will install a separate EC2 for configuration management, install Ansible, and run playbook experiments there. Lastly we will create a SSL certifaction automation for our Webapp.

    There are lots of things to do, but it is fun, right?

    Technology stacks to be used:

    AWS: It will be use for the Server side

    Foreman: It will be used for Centralized Patch Managemant System for our Server Farm

    Ubuntu: Not a big fan of Windows :D

    Kubernetes: It will be used for to manage containers

    Docker: Our application will work on Docker Containers

    Docker Hub: We Will keep our Docker Images in Docker Hub

    Github: We Will store our codes

    Jenkins: We will use for CI process

    Nginx: We will use for Reverse Proxy

    Terraform: Provisioning for our Infrastructure
