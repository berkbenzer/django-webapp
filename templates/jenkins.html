Jenkins Pipeline

  • This section describes how to get started with creating your Pipeline project in Jenkins and introduces you to the various ways that a Jenkinsfile can be created and stored.
  • Step 1: Code Manually push to GitHub Repo
  • Step 2: Jenkins take the Code and Build it as Docker Image.
  • Step 3: Docker Image push to DockerHub Repo.
  • Step 4: The Image download to the Deployment Ubuntu Server.
  • Step 5: Docker container will be started.




    • Step - 1 Jenkins Installation

    Detailed document can be found in DigitalOcean.

    Terminal Section
    $ sudo wget -O /usr/share/keyrings/jenkins-keyring.asc \
    https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key
    $ echo "deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc]" \
    https://pkg.jenkins.io/debian-stable binary/ | sudo tee \
    /etc/apt/sources.list.d/jenkins.list > /dev/null
    $ sudo apt-get update
    $ sudo apt-get install jenkins




    Step - 2 Jenkins Configuration

  • To set up Jenkins, go to your browser and open http://localhost:8080 where Jenkins server is running.
  • The Unlock Jenkins screen will appear as shown below, asking for the administrator password.



  • In terminal run the command below paste the output to the Unlock Jenkins section

    • Terminal Section
    $ sudo cat /var/lib/jenkins/secrets/initialAdminPassword
  • The next screen display the plugins section. We will move on with the suggested plugins.

  • The installation process will begin.


  • Confirm either the domain name for your server or your server’s IP address

  • Click Save and Finish. You’ll receive a confirmation page confirming that “Jenkins is Ready!”

    • Step - 2 Create Pipeline

    Detailed document can be found in.

  • Login to Jenkins


  • From the Jenkins home page (i.e. the Dashboard of the Jenkins classic UI), click New Item at the top left.
  • In the Enter an item name field, specify the name for your new Pipeline project. Caution: Jenkins uses this item name to create directories on disk. It is recommended to avoid using spaces in item names, since doing so may uncover bugs in scripts that do not properly handle spaces in directory paths.


  • Scroll down and click Pipeline, then click OK at the end of the page to open the Pipeline configuration page (whose General tab is selected).
  • Configure the git repo.


  • Click the Pipeline tab at the top of the page to scroll down to the Pipeline section.


  • And Save the configuration

    • Click the Pipeline tab at the top of the page to scroll down to the Pipeline section. Note: If instead you are defining your Jenkinsfile in source control, follow the instructions in In SCM below.



    Step - 3 Build



  • Click the Job name at the Dashboard click the Build button on the right side


  • In this section defined pipeline with Jenkisfile can be on Stage View.


  • For the Build logs click to the related build number and then select the Console output.


  • You can display the logs in Console Output section.
  • You can display the pipeline created a Docker images and start the Docker container.
